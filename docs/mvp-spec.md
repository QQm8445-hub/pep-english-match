# 人教PEP版英语消消乐 — MVP需求文档

**项目名称：** 人教 PEP 版英语消消乐
**文档版本：** v1.0（MVP版）
**撰写日期：** 2026-04-03
**适用年级：** 小学 3～6 年级（MVP 先做三年级）
**开发形式：** 纯 HTML 单文件网页版

---

## 一、MVP 目标

> MVP（最小可行产品）目标：用最少的开发量，跑通完整游戏闭环，验证核心玩法可行性，之后再逐步扩展。

**MVP 核心验证点：**
- 消消乐核心玩法可玩、流畅
- 单词卡片展示 + 点击消除逻辑正确
- 本地数据存储可用（分数、排行榜）
- 界面适配手机 + 电脑

**MVP 范围：三年级 Unit 1 一个关卡跑通完整流程**

---

## 二、MVP 功能范围（精简版）

### ✅ 纳入 MVP

| 功能模块 | 说明 |
|---|---|
| 昵称登录 | 输入昵称即可开始，本地 localStorage 存储 |
| 年级/单元选择 | MVP 仅展示三年级 Unit 1，结构预留扩展 |
| 游戏棋盘 | 5×5 网格，单词卡片随机填充 |
| 消除玩法 | 点击两张相同单词卡片 → 消除全场同类单词 |
| 步数限制 | 每局 25 步，步数归零游戏结束 |
| 计分系统 | 消除一个单词 +5 分，达到 150 分通关 |
| 单词发音 | 消除时播放单词发音（Web Speech API） |
| 排行榜 | 本地存储前 10 名，按分数排序 |
| 通关/失败提示 | 弹窗展示结果 + 重玩按钮 |
| 基础卡通 UI | 圆角卡片、渐变背景、大字体、明亮配色 |
| 手机适配 | 响应式布局，手机可正常操作 |

### ❌ 暂不纳入 MVP（后续版本）

| 功能 | 推迟原因 |
|---|---|
| 3-6年级全册单词库 | 数据量大，MVP 验证后再填充 |
| 游戏道具（炸弹/火箭/魔力球） | 核心玩法验证后再加趣味层 |
| 错题本 | 需要发音识别判断，逻辑复杂 |
| 家长统计页 | MVP 阶段用排行榜替代 |
| 背景音乐 | 非核心体验 |
| 消除动画特效 | MVP 先做功能，动效后续优化 |
| 微信小程序嵌入 | 网页版跑通后再适配 |

## 三、页面结构（MVP 共 3 个页面）

```
首页（登录 + 开始）
  → 游戏页（棋盘 + 操作）
    → 结算页/弹窗（通关/失败 + 排行榜）
```

### 3.1 首页
- 游戏标题（大字，卡通风格）
- 昵称输入框（placeholder：输入你的名字）
- 开始游戏按钮
- 查看排行榜入口
- 年级/单元选择（MVP 固定显示「三年级 Unit 1」，按钮置灰提示"更多关卡即将开放"）

### 3.2 游戏页
- 顶部信息栏：昵称 | 步数剩余 | 当前分数
- 5×5 单词棋盘（共 25 格）
- 单词卡片：展示英文单词 + 可爱图标/颜色区分
- 底部：返回首页按钮
- 点击逻辑：
  - 第一次点击：卡片高亮选中
  - 第二次点击同类单词：消除全场同类，播放发音，计分，步数-1
  - 第二次点击不同单词：切换选中目标，步数-1
  - 步数=0 且 分数<150：触发失败弹窗
  - 分数≥150：触发通关弹窗

### 3.3 结算弹窗
- 通关：🎉 图标 + "太棒了！" + 本局得分 + 重玩 / 返回首页
- 失败：😢 图标 + "再试一次！" + 本局得分 + 重玩 / 返回首页
- 自动将得分写入排行榜

### 3.4 排行榜页（弹窗或独立页）
- 显示前 10 名：排名 | 昵称 | 分数
- 关闭/返回按钮

## 四、核心玩法逻辑说明

### 4.1 棋盘初始化
- 从当前单元单词库（10～12 个单词）中随机抽取，重复填充 25 格
- 保证每种单词至少出现 2 次（否则无法消除）
- 随机打乱排列

### 4.2 消除规则
1. 玩家点击一张卡片 → 该卡片进入「选中」状态（高亮边框）
2. 玩家再次点击任意卡片：
   - **相同单词** → 消除棋盘上所有该单词的卡片，格子变空，步数-1，分数+(消除数量×5)，播放该单词发音
   - **不同单词** → 取消前一个选中，选中新卡片，步数-1
3. 消除后空格不补充新方块（保留空格，棋盘逐渐稀疏）
4. 当步数=0：检查分数，触发结算

### 4.3 胜负判定
- 分数 ≥ 150 分 → 通关（无论步数是否用完）
- 步数 = 0 且 分数 < 150 → 失败

### 4.4 步数消耗
- 每次点击第二张卡片（无论是否消除成功）消耗 1 步

## 七、技术方案

| 项目 | 方案 |
|---|---|
| 开发语言 | HTML5 + CSS3 + 原生 JavaScript（ES6+） |
| 打包形式 | 单文件 index.html（CSS/JS 内嵌） |
| 数据存储 | window.localStorage |
| 单词发音 | Web Speech API（SpeechSynthesisUtterance，免费，无需音频文件） |
| UI 框架 | 无框架，纯 CSS 实现（减少依赖） |
| 字体 | Google Fonts 或系统字体（圆体，儿童友好） |
| 运行环境 | 浏览器直接打开，无需服务器 |

## 八、UI 设计规范（MVP 基础版）

### 配色方案
- 主背景：天空蓝渐变（#87CEEB → #E0F7FA）
- 卡片颜色：10种柔和马卡龙色（每个单词对应一种颜色，便于快速识别）
- 文字：深色（#333），高对比度
- 按钮：橙色/黄色主色调，圆角，带阴影

### 字体大小
- 单词卡片：18～22px，加粗
- 标题：28～36px
- 按钮：16～20px

### 圆角规范
- 卡片：12px
- 按钮：24px（胶囊形）
- 弹窗：20px

## 九、MVP 验收标准

| 验收项 | 标准 |
|---|---|
| 昵称登录 | 输入昵称后可进入游戏，刷新后昵称保留 |
| 棋盘初始化 | 25格随机填充，每种单词≥2次出现 |
| 消除逻辑 | 点击两次相同单词，全场该单词消除，计分正确 |
| 步数扣减 | 每次第二次点击扣1步，归零触发结算 |
| 发音播放 | 消除时正确播放英文单词发音 |
| 通关判定 | 分数达150分触发通关弹窗 |
| 失败判定 | 步数归零且未达150分触发失败弹窗 |
| 排行榜 | 结算后自动记录，排行榜显示前10名，按分数排序 |
| 手机适配 | iPhone/Android 主流机型可正常游玩，卡片不变形 |
| 无需服务器 | 直接双击 index.html 可运行 |
